# Improved User Search

## Статус
Not Started (Milestone 4)

## Описание
Поиск пользователей по подстроке display name (вместо точного Matrix ID), с комбинированными результатами из локальных контактов и серверного user directory.

## Текущее состояние в FluffyChat
- `new_private_chat` — поле поиска, вызывает `searchUsers` из Matrix SDK
- Требует ввода полного Matrix ID (`@user:server.com`) или точного display name
- Серверный API `/user_directory/search` уже поддерживает partial matching — клиент просто не использует его оптимально

## Точки входа в код
- `lib/pages/new_private_chat/new_private_chat.dart` — контроллер поиска
- `lib/pages/new_private_chat/new_private_chat_view.dart` — UI результатов

## Задачи
- [ ] Нормализация ввода: текст без `@` и `:` → отправлять как поисковый запрос (не интерпретировать как Matrix ID)
- [ ] Серверный поиск через `client.searchUserDirectory(term)` — убедиться что partial matching работает
- [ ] Параллельный локальный поиск: фильтрация известных комнат/пользователей по подстроке
- [ ] Объединённый UI: сначала локальные контакты, затем серверные результаты (визуальное разделение секций)
- [ ] Debounce на поисковый ввод (не отправлять запрос на каждый символ)

## Зависимости
- **Серверные:** Synapse должен корректно индексировать display names в user directory
- **Пакеты:** нет дополнительных зависимостей

## Открытые вопросы
- Нужен ли fuzzy search или достаточно substring matching?
- Как обрабатывать кириллицу в поиске (нормализация регистра)?
