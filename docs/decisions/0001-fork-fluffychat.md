# ADR-0001: Fork FluffyChat as Base for FluffyX

## Статус
Accepted

## Контекст
Нужен кастомный Matrix-клиент для корпоративного использования (~50 пользователей) с Telegram-подобным UX. Три варианта:

1. **Написать с нуля** — полный контроль, но огромные трудозатраты (Matrix SDK, E2EE, синхронизация, UI)
2. **Форкнуть Element** — самый популярный клиент, но написан на React Native (Element X) или Kotlin/Swift (нативные), сложная кодовая база
3. **Форкнуть FluffyChat** — Flutter/Dart, компактная кодовая база (~25-30k строк), хороший Matrix Dart SDK от Famedly

## Решение
Форкнуть FluffyChat v2.5.1 и расширять его как FluffyX.

Причины:
- **Flutter** — единая кодовая база для Android и iOS, быстрая итерация
- **Компактность** — легко разобраться и модифицировать
- **Matrix Dart SDK** — зрелая библиотека с E2EE (Vodozemac), VoIP, синхронизацией
- **Существующий VoIP код** — `VoipPlugin`, `Dialer` уже реализованы (за флагом `experimentalVoip`)
- **Active development** — FluffyChat активно поддерживается, можно подтягивать upstream обновления

## Последствия

### Плюсы
- Быстрый старт: базовый мессенджер уже работает
- E2EE, синхронизация, push — всё из коробки
- Один язык (Dart) для всей логики

### Минусы
- **AGPL-3.0** — обязаны оставаться open source, предоставлять исходники пользователям
- Привязка к архитектуре FluffyChat (controller+view, Provider)
- Upstream обновления могут конфликтовать с кастомным кодом
- Необходимость минимизировать инвазивные изменения для облегчения мержей

### Митигация рисков upstream конфликтов
- Все модификации помечаются `// FluffyX: <description>`
- Новый функционал выносится в отдельные файлы где возможно
- Документируются все точки модификации upstream кода
